Efficiency

Here is the only function we have for handling all page navigation.

(function( buttons, pages ){
	var index = buttons.length;
	while( index-- ){
		// register on-click event for all buttons
		buttons[ index ].onclick = function( event ){
			// first remove all "active-button"
			var index = buttons.length;
			while( index-- ){
				buttons[ index ].classList.remove( "active-button" );
				pages[ index ].classList.remove( "active-page" );
			}
			// then add "active-button" to the target
			this.classList.add( "active-button" );
			// then add "active-page" to the target according to activated button
			document.getElementById( "page-" + this.id.substr( 7 ) ).classList.add( "active-page" );
		}
	}
}( document.querySelectorAll( "span[id|=button]" ), document.querySelectorAll( "div[id|=page]" ) ) );

Also the same effect could be done, with just 2 lines of pure CSS. See our github repository.

Formal definition

User Experience simply refers to the way a product behaves and is used in the real world. A positive user experience is one in which the goals of the both user and the organization that created the product are met.

Jesse James Garret. Author of The Elements of Users Experience

Human Errors

Design in industry should not be just about adorning or beautifying the product. It also should be about preventing users from making mistakes or errors.

Mostly human errors is because of poorly design system.

Black Box Thinking by Matthew Syed.

For Example:

Mismatch:

  • Trying to fit desktop-box-model into a smartphone screen-size is a mismatch.

Match:

  • What smartphones users really need is a separate and different layout then minor-desktop-view.

Poorly Design System:

  • The present layout for responsive design causes headache for both users and businesses.

Why User Experience Design

studies of user behavior on the Web find a low tolerance for difficult designs or slow sites.

People do not want to wait.

And they don't want to learn how to use a home page. There's no such thing as a training class or a manual for a Web site.

People have to be able to grasp the functioning of the site immediately after scanning the home page—for a few seconds at most.

Otherwise, most casual users simply leave the site and browse or shop elsewhere. wikipedia

Jakob Nielson
